# UC-NS1: Управление подпиской и подключением клиента

**Актор:** Клиент (Frontend / Mobile App)

## Цель
Клиент подключается к системе уведомлений и становится доступным для получения сообщений.

## Предусловия
- Клиент авторизован и имеет действительный токен.
- Notification Service запущен и доступен.

## Основной сценарий
1. Клиент устанавливает соединение с Notification Hub (SignalR/WebSocket).
2. Передаёт свой идентификатор (userId) для привязки соединения.
3. Сервис регистрирует соединение и присваивает клиента к его каналу/группе.
4. Клиент готов получать уведомления в реальном времени.

## Альтернативные сценарии
- **A1:** Соединение не установлено → попытка переподключения.
- **A2:** Пользователь не авторизован → отказ в подключении.
- **A3:** Клиент отключается → соединение удаляется.

## Постусловия
- Клиент находится в системе подписки (до отключения).
- Канал доставляет сообщения клиенту по userId.
