# UC-CL2: Просмотр каталога

**Актор:** Покупатель  
**Цель:** Просмотр списка товаров по категориям и фильтрам.

## Предусловия
- Категории и товары созданы и доступны.

## Основной сценарий
1. Пользователь открывает каталог.
2. UI запрашивает у API Gateway список категорий.
3. Gateway переадресует запрос в Catalog Service.
4. Пользователь выбирает категорию.
5. Сервис возвращает список товаров с пагинацией (например, 20 на страницу).
6. Пользователь может применить фильтры (цена/рейтинг/бренд и т.п.).

## Альтернативные потоки
- **5а. Категория пустая** → отображается сообщение "Нет товаров".
- **6а. Ошибочный фильтр** → возвращается базовый список.

## Постусловия
- Пользователь получает и просматривает товары.

## Бизнес-правила
- Пагинация обязательна для оптимизации работы базы.
