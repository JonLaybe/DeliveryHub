# UC-OR2: Отслеживание статуса заказа

**Актор:** Покупатель  
**Цель:** Получить текущее состояние заказа.

## Предусловия
- Заказ существует.
- Пользователь имеет доступ к заказу (владелец).

## Основной сценарий
1. Пользователь открывает раздел "Мои заказы".
2. UI отправляет запрос `/order/{id}/status`.
3. Order Service возвращает статус, например:
   - CREATED
   - PROCESSING
   - PACKED
   - IN_DELIVERY
   - COMPLETED
4. UI отображает статус и дополнительную информацию.

## Альтернативные потоки
- **2а. Неверный ID** → ошибка *"Заказ не найден"*
- **2б. Нет доступа** → *"Недостаточно прав"*
- **3а. Сервис недоступен** → вернуть кешированные данные (если есть)

## Постусловия
- Пользователь получает текущий статус.

## Нефункциональные требования
- Рекомендуется обновление статуса в реальном времени через события (RabbitMQ/Kafka/WebSockets).
